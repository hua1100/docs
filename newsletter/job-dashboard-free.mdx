找工作是一場耐力賽，想即時掌握求職市場動態，不能靠每天手動刷求職網站，不僅容易漏看，更會產生「我是不是錯過了什麼好機會」的資訊焦慮。

我長期關注 AI 相關職缺，遠端更是我相當在意的工作條件，這次製作了兩個儀表板：

- **行銷/PM職類的遠端工作儀表板**
- **數位轉型/流程自動化/AI工程師職缺儀表板**

讓我能每週掌握三個重要趨勢：

1. **遠端政策公司名單**：每週關注哪些公司釋出遠端職缺，而不是急著跳船時才開始建立名單
2. **薪資分布**：抓出平均月薪與薪資分布圖，了解職缺在市場上的價碼，實際談薪水時更有底氣
3. **技能趨勢**：統計出熱門技能排行，持續充實相關技能，往夢幻職缺努力

## 做法

### A. 爬資料、生成儀表板

104 的爬蟲，網路上有很多公開資源，下載就能直接用（或讓 AI 幫你用）。

要呈現什麼數據、儀表板風格是什麼？這點可以跟 AI 多做討論，要讓 AI 懂你想要的設計，丟關鍵字是最有效率的方式。例如這次的儀表板就是用「salesforce lightning」這個關鍵字讓 AI 生成，效果相當不錯。

### B. 部署到 GitHub Pages

到 GitHub 建立一個倉庫，把程式碼 push 上去，接著在 GitHub 的 Settings → Pages，將 Source 設定為 Deploy from a branch，並選擇 main 分支，就會獲得一個可以公開分享的網站連結。

### C. 設定 GitHub Action，每週觸發爬蟲更新資料

跟 AI 說你要建立 GitHub Action，每週自動更新職缺資料。AI 此時會建立一個名為 `.github/workflows` 的資料夾，資料夾裡面放一個 `.yml` 檔，這個檔案就是給 GitHub Actions 看的「自動化劇本」。

用「廚房做菜」的比喻來解釋這個 SOP 的結構：

| 術語 | 意思 |
|------|------|
| Workflow | 整張 SOP 說明書的標題（例如：「每週自動更新職缺菜單」） |
| On | 什麼時候開始做這道菜？ |
| Jobs | 要做哪些大項的工作（例如：「備料並烹飪」） |
| Steps | 每個任務裡的細節動作，按順序執行（1.洗菜 2.切菜 3.炒菜...） |

<Tip>
其實只要複製「建立 Github action，每週自動更新職缺資料。」這句話給 AI，它就能幫你完成所有事情。了解越多細節，對專案的掌控度越高，也會大大減少被 AI 牽著鼻子走而不自知的風險。
</Tip>
